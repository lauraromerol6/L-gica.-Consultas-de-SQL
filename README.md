# Lógica. Consultas de SQL.
### Descripción
Este proyecto se centra en analizar el supuesto funcionamiento de un local de alquileres de películas. Para ello, se utiliza la base de datos Shakila. Asimismo, a través de consulas SQL en PostgreSQL, se analizan distintos aspectos relacionados con clientes, películas, actores y alquileres.

### Estructura del proyecto
En este caso, se va a proceder a explicar la estructura de la base de datos de Shakila, la cual está compuesta por varias tablas relacionadas:
- **Actor**: información de los actores
- **Film**: datos de las películas
- **Film_actor**: relación entre actores y películas
- **Category**: categorías de las películas
- **Film_category**: relación entre películas y categorías
- **Customer**: datos de los clientes
- **Rental**: registros de alquileres
- **Inventory**: copias disponibles de cada película

### Instalación
Para este proyecto es necesario instalarse la base de datos de Shakila y para proceder al análisis se deberá descargar Dbeaver y PostgreSQL.

### Conclusión
En concluión, Shakila permite interpretar casos reales de negocio para aprender a utilizar SQL de forma práctica, adquiriendo conocimientos de análisis de datos, realización de consultas y derivados y de modelado relacional.
